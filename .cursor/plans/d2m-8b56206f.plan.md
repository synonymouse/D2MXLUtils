# D2MXLUtils UI Redesign Plan

## Суть проекта

D2MXLUtils — утилита для Diablo 2 LOD MedianXL, создающая продвинутый лутфильтр с текстовым редактором правил (заимствованным из D2Stats), оверлеем уведомлений и возможностью интеграции с нативным лутфильтром игры. Цель — дать опытным игрокам мощный, выразительный и удобный инструмент без "generic website" эстетики.

## Фаза 3: Редактор правил (Loot Filter)

### 3.1 Формат правил D2Stats

Заимствуем текстовый формат из D2Stats ([docs/index_d2Stats.md](docs/index_d2Stats.md)):

```
# Comment line
"Item Name" {stat regex}
"Ring$" rare {[3-5] to All Skills}
"Amulet" unique {Enemy Cold}
```

Элементы синтаксиса:

- `# ...` — комментарий
- `"Item Name"` — паттерн имени предмета (regex)
- `rare`, `unique`, `set`, `magic`, `crafted` — модификаторы качества
- `eth` — ethereal модификатор
- `{...}` — regex для матчинга статов

### 3.2 Парсер правил (Rust backend)

- Модуль `src-tauri/src/text_rules.rs`:
  - Парсинг текстового формата в структуру `TextRule`
  - Компиляция regex-паттернов
  - Конвертация в существующий `Rule` формат
- Tauri-команды: `parse_rules`, `validate_rules`, `compile_filter`

### 3.3 CodeMirror 6 интеграция

- Установить `@codemirror/view`, `@codemirror/state`, `@codemirror/language`
- Создать `src/editor/`:
  - `d2rules-language.ts` — определение грамматики для подсветки:
    - Строки в кавычках (оранжевый/золотой)
    - Фигурные скобки и их содержимое (regex — зелёный)
    - Комментарии (серый)
    - Ключевые слова качества (цвета качества из игры)
  - `d2rules-autocomplete.ts` — автодополнение
  - `RuleEditor.svelte` — Svelte-обёртка вокруг CodeMirror

### 3.4 Autocomplete

Источники данных для автодополнения:

- Список базовых предметов (из `Items.txt` или хардкод популярных)
- Названия уникальных предметов
- Названия статов (Strength, Vitality, All Skills, etc.)
- Ключевые слова (`rare`, `unique`, `eth`, `show`, `hide`, `notify`)

### 3.5 Система профилей

- Профиль = отдельный `.d2rules` файл
- UI: выпадающий список профилей, кнопки New/Rename/Delete/Duplicate
- Tauri-команды: `list_profiles`, `load_profile`, `save_profile`, `delete_profile`
- Хранение профилей в `%APPDATA%/D2MXLUtils/profiles/`

### 3.6 Звуковые эффекты

- Папка `assets/sounds/` с предустановленными звуками
- Возможность добавить свой `.wav`/`.mp3`
- В синтаксисе правил: `"Item" unique {Skills} !sound:alert.wav`
- Tauri-команда: `play_sound`

## Фаза 4: Оверлей и уведомления

### 4.1 Перетаскиваемая область уведомлений

- При зажатом Ctrl область уведомлений становится draggable
- Визуальная индикация (рамка, курсор move)
- Сохранение позиции в настройках
- Привязка к углам/краям (snap)

### 4.2 Стилизация уведомлений

image.png

Вкладка "Notification Style" в главном окне:

- Выбор шрифта (моноширинный по умолчанию)
- Размер текста
- Прозрачность фона
- Длительность показа уведомления
- Направление стека (снизу вверх / сверху вниз)
- Превью уведомления в реальном времени

### 4.3 Подсветка статов в уведомлениях

Заимствуем из D2Stats логику форматирования:

- Парсинг строки статов
- Цветовое выделение ключевых статов (Skills — золотой, Resistance — по стихии, etc.)
- Формат: `+3 to All Skills` → `+3` зелёный, `to All Skills` золотой
