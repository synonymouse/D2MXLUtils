# D2MXLUtils UI Redesign Plan

## Суть проекта

D2MXLUtils — утилита для Diablo 2 LOD MedianXL, создающая продвинутый лутфильтр с текстовым редактором правил (заимствованным из D2Stats), оверлеем уведомлений и возможностью интеграции с нативным лутфильтром игры. Цель — дать опытным игрокам мощный, выразительный и удобный инструмент без "generic website" эстетики.

---

## Архитектура форматов правил

```
┌─────────────────────────────────────────────────────────────┐
│                      ФОРМАТЫ ПРАВИЛ                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [Пользователь в UI]                                       │
│        │                                                    │
│        ▼                                                    │
│   ┌─────────────┐                                           │
│   │  DSL Editor │  ← CodeMirror, человекочитаемый формат    │
│   │  (UI)       │    "Ring$" unique {Skills}                │
│   └──────┬──────┘                                           │
│          │                                                  │
│          ▼          ▲                                       │
│   ┌──────────────────────┐                                  │
│   │  DSL ↔ JSON Parser   │  ← rules/dsl.rs ✅               │
│   │  parse_dsl()         │                                  │
│   │  to_dsl()            │                                  │
│   └──────────────────────┘                                  │
│          │          ▲                                       │
│          ▼                                                  │
│   ┌─────────────┐                                           │
│   │    JSON     │  ← Хранение, совместимость с игрой        │
│   │  (rules.rs) │    FilterConfig, Rule структуры ✅        │
│   └─────────────┘                                           │
│          │                                                  │
│          ├──────────► Экспорт в игровой фильтр MedianXL     │
│          └──────────► Импорт из игры                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**JSON** — единственный формат хранения правил:

- Совместим с игровым лутфильтром MedianXL (для импорта/экспорта)
- Структуры `Rule`, `FilterConfig` в `rules/mod.rs`
- Расширен для поддержки всех фич DSL

**DSL** — формат для UI/редактора:

- Человекочитаемый синтаксис D2Stats
- Парсится в JSON при сохранении
- Генерируется из JSON при загрузке

## Фаза 3: Редактор правил (Loot Filter)

### 3.1 Формат правил DSL (D2Stats-совместимый) ✅

Синтаксис редактора (отображается пользователю):

```
# Комментарий
"Item Name" {stat regex}
"Ring$" rare {[3-5] to All Skills}
"Amulet" unique {Enemy Cold}
"." sacred eth gold sound1
```

Элементы синтаксиса:

- ✅ `# ...` — комментарий
- ✅ `"Item Pattern"` — паттерн имени предмета (regex)
- ✅ **Quality:** `low`, `normal`, `superior`, `magic`, `set`, `rare`, `unique`, `craft`, `honor`
- ✅ **Tier:** `0`, `1`, `2`, `3`, `4`, `sacred`, `angelic`, `master`
- ✅ **Misc:** `eth` — ethereal модификатор
- ✅ `{...}` — regex для матчинга статов
- ✅ **Color:** `transparent`, `white`, `red`, `lime`, `blue`, `gold`, `grey`, `black`, `pink`, `orange`, `yellow`, `green`, `purple`, `hide`, `show`
- ✅ **Sound:** `sound_none`, `sound1`-`sound6`
- ✅ **Display:** `name`, `stat`

### 3.2 Рефакторинг rules.rs → модуль rules/ ✅

✅ Разбито на модули:

```
src-tauri/src/rules/
├── mod.rs          # Rule, FilterConfig, enums (Quality, Tier, Color...)
├── matching.rs     # MatchContext, regex-матчинг
└── dsl.rs          # DSL ↔ JSON парсер
```

✅ **Расширена структура `Rule`** для поддержки всех фич DSL:

```rust
pub struct Rule {
    // Существующие поля (совместимость с игрой)
    pub active: bool,
    pub rule_type: i32,
    pub item_quality: i32,
    pub ethereal: i32,
    pub params: RuleParams,
    pub show_item: bool,
    pub notify: bool,
    pub automap: bool,
    
    // Новые поля для DSL:
    pub name_pattern: Option<String>,  // regex паттерн имени
    pub stat_pattern: Option<String>,  // regex паттерн для статов
    pub tier: Option<i32>,             // sacred/angelic/master/0-4
    pub color: Option<String>,         // цвет уведомления
    pub sound: Option<u8>,             // номер звука 1-6
    pub display_name: bool,            // флаг name
    pub display_stats: bool,           // флаг stat
    pub source_line: Option<String>,   // исходная DSL строка
}
```

✅ **DSL парсер** в `rules/dsl.rs`:

```rust
/// Парсинг DSL текста в FilterConfig
pub fn parse_dsl(text: &str) -> Result<FilterConfig, Vec<ParseError>>;

/// Конвертация FilterConfig обратно в DSL
pub fn to_dsl(config: &FilterConfig) -> String;

/// Валидация DSL без полного парсинга (для редактора)
pub fn validate_dsl(text: &str) -> Vec<ValidationError>;
```

✅ **Tauri-команды** (зарегистрированы в `main.rs`):

- `parse_filter_dsl(text: String) -> Result<FilterConfig, Vec<ParseError>>`
- `filter_to_dsl(config: FilterConfig) -> String`
- `validate_filter_dsl(text: String) -> Vec<ValidationError>`

✅ **Создан модуль `src/editor/`:**

```
src/editor/
├── index.ts              # Экспорты модуля

```

✅ **d2rules-language.ts** — токенизатор:

- Строки в кавычках (`"..."`) — зелёный
- Stat patterns в фигурных скобках (`{...}`) — regexp стиль
- Комментарии (`# ...`) — серый
- Ключевые слова качества с цветами из игры:
  - `unique` — золотой (#c7b377)
  - `set` — зелёный (#00ff00)
  - `rare` — жёлтый (#ffff00)
  - `magic` — синий (#6969ff)
- Tier, color, sound, display keywords — отдельные стили

✅ **d2rules-theme.ts** — темы:

- Dark и Light темы с CSS-переменными
- `qualityHighlighting` — подсветка качества предметов
- Стилизация lint gutter и tooltips

✅ **d2rules-linter.ts** — единый линтер:

- Интеграция с `validate_filter_dsl` (Tauri backend)
- Интеграция с `parse_filter_dsl` для подсчёта правил
- Debounce 500ms после остановки ввода
- Callback `onResult` для уведомления родителя о результатах
- Очистка диагностик при начале ввода (не мешает печатать)

✅ **RulesEditor.svelte** — Svelte-обёртка:

- Two-way binding для `value`
- Callbacks: `onchange`, `onsave` (Ctrl+S), `onvalidate`
- History (undo/redo), bracket matching, active line highlight
- Lint gutter с иконками ошибок/предупреждений

✅ **LootFilterTab.svelte** — интеграция:

- Статус валидации в шапке (✓ N rules / ✗ N errors)
- Кнопки Reset / Save
- Syntax Reference (справка по синтаксису)
