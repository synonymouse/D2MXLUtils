## Ключевые находки

### 1. MedianXL (D2Sigma.dll) переопределяет рендеринг

**Важнейшее открытие:** В MedianXL отрисовка тултипов предметов происходит через `D2Sigma.dll`, а НЕ напрямую через `D2Client.dll`.

Call stack при отображении тултипа предмета:
```
D2Sigma.dll+F5755      <- отрисовка текста (D2Win)
D2Sigma.dll+95607      <- строковая функция
D2Sigma.dll+F9353      <- основной рендер-цикл
D2Client.dll+44380     <- вызов из D2Client
D2Client.dll+44E74
D2Client.dll+3CB77
```

### 2. Функция глобальной отрисовки тултипов

**Адрес:** `D2Sigma.dll.text+1369D0`

**Исходный файл:** `..\ui\Loot.cpp` (видно из отладочных строк)

**Поведение:**
- Если заменить первый байт `A1` на `C3` (ret) — **все тултипы предметов исчезают**
- Функция вызывается для каждого тултипа на экране
- Получает `ECX = указатель на внутреннюю структуру тултипа` (НЕ pUnit!)

**Код функции:**
```asm
D2Sigma.dll.text+1369D0 - A1 F453F80B           - mov eax,[D2Sigma.dll+6C53F4]
D2Sigma.dll.text+1369D5 - 83 38 03              - cmp dword ptr [eax],03
D2Sigma.dll.text+1369D8 - 0F85 02000000         - jne D2Sigma.dll.text+1369E0
D2Sigma.dll.text+1369DE - C3                    - ret
; ... далее основной код отрисовки ...
```

**Важно:** Проверка `[eax] == 3` — это глобальный флаг состояния игры. Если установить `3`, экран становится чёрным (отключается весь рендеринг мира).

### 3. Встроенная настройка MedianXL

Найдена строка: `"display_show_loot_toggle"`

**Адрес в коде:** `D2Sigma.dll.text+F8319`

Это глобальный переключатель показа/скрытия ВСЕХ тултипов лута. Работает на все предметы сразу, не подходит для фильтрации конкретных предметов.

### 4. Функция проверки типа предмета (не влияет на рендеринг)

**Адрес:** `D2Sigma.dll.text+C4400`

**Параметры:** `ECX = pUnit`, `EDX = маска`

```asm
D2Sigma.dll.text+C4400 - test ecx,ecx          ; pUnit != NULL?
D2Sigma.dll.text+C4402 - je C4416              ; если NULL -> return 0
D2Sigma.dll.text+C4404 - cmp dword ptr [ecx],04 ; [pUnit] == 4 (ITEM)?
D2Sigma.dll.text+C4407 - jne C4416             ; если не ITEM -> return 0
D2Sigma.dll.text+C4409 - mov eax,[ecx+14]      ; eax = pUnitData
D2Sigma.dll.text+C440C - test eax,eax          ; pUnitData != NULL?
D2Sigma.dll.text+C440E - je C4416
D2Sigma.dll.text+C4410 - mov eax,[eax+18]      ; eax = [pUnitData+0x18]
D2Sigma.dll.text+C4413 - and eax,edx           ; применить маску
D2Sigma.dll.text+C4415 - ret
D2Sigma.dll.text+C4416 - xor eax,eax           ; return 0
D2Sigma.dll.text+C4418 - ret
```

**Результат тестирования:** Замена на `ret` в начале НЕ влияет на отображение тултипов. Это вспомогательная функция (вероятно для colorblind режима).

### 5. Код в D2Client.dll

**Адрес:** `D2Client.dll.text+6AFBB` (смещение +0x6AFBB от базы)

**База D2Client.dll:** `0x6FAB0000`

```asm
D2Client.dll.text+6AFBB - cmp dword ptr [edi],04  ; проверка типа юнита
D2Client.dll.text+6AFBE - jne ...                  ; если не ITEM, пропустить
```

**Поведение:** Срабатывает при рендеринге, `EDI = pUnit`. Однако замена `jne` на `jmp` НЕ скрывает тултипы — это код подготовки данных, не отрисовки.

## Выводы и гипотезы

### Архитектура рендеринга тултипов в MedianXL

```
┌────────────────────────────────────────────────────────────────────────┐
│                        Архитектура (гипотеза)                          │
├────────────────────────────────────────────────────────────────────────┤
│  1. D2Client итерирует юниты на земле (D2Client.dll+6AFBB)             │
│  2. Для каждого ITEM вызывает функции D2Sigma.dll                      │
│  3. D2Sigma формирует структуру тултипа (текст, цвет, позиция)         │
│  4. Структура добавляется в очередь отрисовки                          │
│  5. D2Sigma.dll.text+1369D0 рендерит каждый тултип из очереди               │
└────────────────────────────────────────────────────────────────────────┘
```

### Почему pUnit не передаётся в функцию рендеринга

Функция `1369D0` получает готовую структуру тултипа (`ECX`), а не `pUnit`. К моменту вызова этой функции решение "какие предметы показывать" уже принято. Структура содержит:
- Текст тултипа
- Координаты отрисовки
- Цвет/стиль
- Возможно ссылку на pUnit (нужно исследовать)

### Где искать точку хука

**Нужно найти код ВЫШЕ по стеку**, где:
1. Итерируются предметы на земле
2. Принимается решение "добавить в очередь рендеринга или нет"
3. Формируется структура тултипа

Кандидаты для исследования:
- `D2Sigma.dll.text+F9353` — основной рендер-цикл
- `D2Client.dll+44380` — вызов D2Sigma из D2Client
- Код формирования очереди тултипов

## Методы поиска (использованные)

### 1. Hardware Breakpoint на pUnit

1. Залогировать `pUnit` из D2MXLUtils (добавить debug log в notifier.rs)
2. В CE добавить адрес pUnit
3. `Find out what accesses this address`
4. Искать адреса из D2Client.dll и D2Sigma.dll

### 2. Поиск через строку тултипа

1. Выбросить предмет с уникальным названием
2. Найти текст тултипа в памяти (UTF-16)
3. `Find out what accesses this address`
4. Найти функции записи/чтения строки

### 3. Трассировка Call Stack

1. Поставить breakpoint на функцию D2Win (отрисовка текста)
2. Посмотреть Call Stack
3. Подняться по стеку до D2Sigma/D2Client

## Дальнейшие шаги

### Вариант A: Исследовать структуру тултипа

1. Поставить breakpoint на `D2Sigma.dll.text+1369D0`
2. Исследовать структуру по адресу `ECX`:
   - Искать указатель на pUnit внутри структуры
   - Найти смещение где хранится pUnit
3. Если pUnit найден — добавить проверку в начале функции

### Вариант B: Найти место формирования очереди

1. Поставить breakpoint на `D2Sigma.dll+F9353` (рендер-цикл)
2. Трассировать код выше
3. Найти где предметы добавляются в очередь рендеринга
4. Хукнуть это место для фильтрации

### Вариант C: Хукнуть D2Client перед вызовом D2Sigma

1. Исследовать `D2Client.dll+44380`
2. Найти где передаётся pUnit в D2Sigma
3. Добавить проверку флага скрытия перед вызовом

## Полезные адреса

### D2Sigma.dll (MedianXL)

| Смещение | Описание |
|----------|----------|
| `+1369D0` | Функция рендеринга тултипа (ret = скрыть все) |
| `+C4400` | Функция проверки типа предмета (ECX=pUnit) |
| `+F9353` | Основной рендер-цикл |
| `+F8319` | Ссылка на "display_show_loot_toggle" |
| `+6C53F4` | Указатель на глобальный флаг состояния |

### D2Client.dll

| Смещение | Описание |
|----------|----------|
| `+6AFBB` | Проверка типа юнита (EDI=pUnit) |
| `+44380` | Вызов функций D2Sigma |

### Структуры

| Структура | Смещение | Поле |
|-----------|----------|------|
| Unit | +0x00 | dwType (4 = ITEM) |
| Unit | +0x14 | pUnitData |
| ItemData | +0x18 | Флаги (читается в C4410) |
| ItemData | +0x48 | iEarLevel (для фильтрации) |
